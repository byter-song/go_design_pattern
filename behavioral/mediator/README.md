# 中介者模式 (Mediator Pattern)

## 概述

中介者模式通过引入一个中心协调者来管理对象之间的交互，从而减少对象之间的直接耦合。

## Go 语言实现特点

- 很适合用接口描述中介者能力
- 同事对象只依赖中介者接口
- 常见于聊天系统、事件协调、UI 组件协作

## 代码示例

当前实现用聊天室演示：

- `ChatRoom` 负责路由消息
- `User` 只把消息交给中介者，不直接持有其他用户

## 使用场景

1. 多对象交互复杂
2. 想减少网状依赖
3. 希望把协调逻辑集中管理
