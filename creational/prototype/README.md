# 原型模式 (Prototype Pattern)

## 概述

原型模式通过复制现有对象来创建新对象，而不是每次都从零初始化。它适用于对象构建成本较高、或者希望从模板对象快速生成变体的场景。

## Go 语言实现特点

- 结构体赋值天然支持浅拷贝
- 深拷贝通常由显式方法完成
- 对切片、Map、指针字段需要特别小心
- Go 没有内建通用深拷贝，通常按业务手写最安全

## 核心概念

```go
copy := *original
```

这会得到一个新的结构体值，但其中的切片、Map 等引用类型字段仍然共享底层数据，因此它只是浅拷贝。

## 代码示例

当前实现的 `Document` 包含以下字段：

- `Tags []string`
- `Metadata map[string]string`
- `Sections []Section`
- `Section.Notes []string`

### 浅拷贝

```go
cloned := original.ShallowClone()
```

修改 `cloned.Tags`、`cloned.Metadata`、`cloned.Sections[0].Notes` 会影响原对象。

### 深拷贝

```go
cloned := original.DeepClone()
```

深拷贝会：

- 复制切片内容
- 复制 map 键值
- 递归复制嵌套结构体里的切片字段

## 深拷贝与浅拷贝在 Go 中的关键区别

### 浅拷贝会共享的内容

- 切片底层数组
- map 底层哈希表
- 指针指向的对象

### 深拷贝需要手动复制的内容

- `append([]T(nil), src...)`
- 遍历 `map` 后重新写入
- 对嵌套对象递归复制

## 使用场景

1. 基于模板快速创建对象
2. 避免重复执行昂贵初始化
3. 需要保留原始配置并派生多个变体

## 优缺点

### 优点

- 创建速度快
- 易于从模板派生多个对象
- 对复杂配置对象很实用

### 缺点

- 深拷贝逻辑容易遗漏引用字段
- 对象层次越深，复制成本越高
- 不恰当的浅拷贝容易引入共享状态 bug

## 与其他模式的关系

- 与 **建造者模式** 不同：建造者逐步构造，原型直接复制现有对象
- 与 **工厂模式** 不同：工厂强调创建入口，原型强调复制已有实例
